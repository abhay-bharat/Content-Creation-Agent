# AIThinkingLab Visual Style Guide

Reference for maintaining consistent, professional visual design across all content.

---

## Visual Philosophy

**Minimal tech aesthetic with purpose.**

Every visual choice should communicate: "This creator is thoughtful, technical, and respects my time."

---

## Color System

### Base Palette
- **Primary Dark**: #0F1419 (almost black, slight blue tone)
- **Secondary Dark**: #1A1F2E (slightly lighter)
- **Neutral Light**: #E8EAED (almost white)
- **Neutral Gray**: #9CA3AF (mid-tone)

### Accent Colors (Pick 1-2)
- **Accent 1**: #00D9FF (cyan - technical, modern)
- **Accent 2**: #FF6B35 (orange - energy, warmth)
- **Accent 3**: #00F0A0 (mint - positive, growth)

Use accent sparingly:
- Callout boxes / highlights
- Key numbers / metrics
- Icons / visual emphasis
- CTA buttons

### Color Application
- **Background**: Primary dark (#0F1419)
- **Text**: Neutral light (#E8EAED)
- **Secondary text**: Neutral gray (#9CA3AF)
- **Highlights**: One accent color max per frame

## Typography

### Font Selections
**Headlines**: 
- Preferred: Inter Bold, Outfit Bold (modern, clean)
- Size: 40-60px (large, readable at phone)
- Weight: 700+
- Line height: 1.2

**Body/Subheading**:
- Preferred: Inter Regular, Roboto (neutral, legible)
- Size: 24-32px
- Weight: 400-500
- Line height: 1.4

**Captions/Small text**:
- Size: 14-18px
- Weight: 400
- Color: Neutral gray

### Typography Rules
- **Maximum 3 font sizes per frame** (headline, body, caption)
- **High contrast always** - Text on background must be readable at phone size
- **Short lines** - Break text into 2-3 word chunks for visual rhythm
- **No lowercase for headlines** - Title case or ALL CAPS only
- **Avoid thin fonts** - 400 weight minimum for readability

## Layout Principles

### Frame Structure (All Formats)
1. **Top third**: Hook or key idea
2. **Middle third**: Supporting content (image/code/diagram)
3. **Bottom third**: CTA or next point

### Spacing
- **Margins**: 40-60px from edges (breathing room)
- **Between elements**: 20-30px (visual grouping)
- **Line spacing**: 1.4-1.6x (readable)

### Visual Hierarchy
- **Dominant element**: Largest, brightest, top placement
- **Secondary elements**: Medium size, supporting color
- **Tertiary elements**: Smallest, neutral gray color
- **Goal**: Viewer understands content priority in <1 second

### Grid System
- Think in thirds (rule of thirds)
- Align elements to invisible grid
- Use consistent spacing (multiples of 20px)

## Reel Frame Specifics

### Dimensions
- **Resolution**: 1080px × 1920px (vertical)
- **Safe area**: 40px margins all sides
- **Duration**: 20-35 seconds

### Frame Design Principles
1. **One core idea per frame** - No clutter
2. **Text as image** - Large, styled, not body copy
3. **Visual hierarchy in <1 sec** - What's the main point?
4. **Motion supports meaning** - Reveals, emphasizes, not decorative
5. **Consistent spacing** - Same margins, gaps, alignment

### Reel Frame Template
```
[40px margin]

[HEADLINE - Large, 1-2 lines max]
[20px gap]
[Support text - 24px, 1 line]

[40px - central visual area]

[Bottom section]
[CTA or supporting idea]
[Source/credit if needed]

[40px margin]
```

### Text Sizing in Reels
- **Headline**: 48-60px (must be readable on phone)
- **Support text**: 28-36px
- **Small text/credit**: 16-20px

### Motion Best Practices
- **Entrance animations**: Fade-in, slide-up (150-300ms)
- **Text reveals**: Stagger for rhythm (each line 100-200ms apart)
- **Attention draws**: Subtle pulse or scale on key numbers
- **Exit**: Fade-out (200ms)
- **Never**: Spinning, bouncing, decorative effects

---

## Carousel Slide Specifics

### Dimensions
- **Size**: 1080px × 1350px (Instagram default)
- **Safe area**: 60px margins
- **Consistency**: Same template for all slides in carousel

### Carousel Structure
**Slide 1 (Hook)**
- Large headline (48-60px)
- Subheading (28px)
- Visual teaser (image/icon)
- CTA: "Swipe for more"

**Slides 2-4 (Content)**
- Headline (36-48px)
- Content (text, diagram, or image)
- Icon or visual supporting the idea

**Slide 5+ (Frameworks/Proof)**
- Clean layout
- Diagram or comparison table
- High contrast for readability

**Last Slide (CTA)**
- Recap headline
- Clear CTA (Save, Share, Follow)
- Contact/link

---

## Icon & Visual Element Guide

### Icon Style
- **Style**: Outlined, minimal (2-3px stroke)
- **Sizing**: 40-80px (visible but not dominant)
- **Color**: Use one accent color
- **Spacing**: 20px clear area around each icon

### Diagram & Code Display
- **Diagrams**: Use simple shapes, clear labels, high contrast
- **Code**: Monospace font (24-32px), syntax highlighting, dark background
- **Trade-off tables**: Clean grid, accent color for key cells
- **Comparison**: Side-by-side with clear visual difference

## Accessibility Checklist

Before publishing:
- [ ] Text readable on small phone screen?
- [ ] Color contrast ≥4.5:1 (WCAG AA standard)?
- [ ] Text: No all-caps for long passages?
- [ ] Captions: Included for video/audio?
- [ ] Alt text: Descriptive alt-text for images?
- [ ] Color blind safe: Not relying only on color to convey meaning?

## Examples of Good AIThinkingLab Visuals

### Minimal, High-Impact
- Dark background, accent color spotlight
- Large, readable text (18px+ on 1080px width = phone readable)
- One core visual element
- Clear information hierarchy
- White space respected

### Engineer-to-Engineer Design
- Looks like a product interface or well-designed system
- Technical precision (diagrams, code, data)
- Not "designed by a marketer"
- Respects audience intelligence

### What NOT to Do
- Clutter with decorations
- Tiny text (unreadable on phone)
- Comic sans, overly playful fonts
- Every color of the rainbow
- Cheesy stock photos
- Generic social media templates
